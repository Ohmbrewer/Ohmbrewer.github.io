How does it work?

Our path to a better easier beer

  • Centralized Web based Application

    Control center for the entire operation

    Ohmbrewer is designed for the web from the ground up. The Ohmbrewer application can be hosted on anything from a small Raspberry Pi unit, to a personal computer, or even a cloud service. The application can be installed on a private or public network - whichever makes the most sense for the user’s purposes.

  • Integrated Hardware

    Hardware OPerating Subsystem (HOPS)

    The Ohmbrewer modules have a central microprocessor that gathers information from temperature probes and sensors, As well as controlling electrical relays to modulate heating elements and power to other peripherals. The HOPS modules are designed to integrate seamlessly with existing brewing hardware configurations and communicates over a wireless network to the centralized controller unit.

  • Rhizome

    The heart of the HOPS module

    The brain and communication center of the hardware is our Rhizome PID controller. It is built on a small Spark microprocessor that is WIFI enabled. We’re designing this to be a drop-in replacement for many PID controllers on the market today. If you have a pre-existing system and do not wish to use the full HOPS set up, just plant it in your existing system and you’re ready to go.

  • WIFI enabled brewing

    remote access to system from any wifi device

    Because the application is served over the network, the user can check on the status of a batch and monitor equipment remotely from a wireless device such as tablets or smart phones. The system will be able to send configurable updates to a phone or email address if given access to the Internet and alert the user of important events during the brewing cycle.
